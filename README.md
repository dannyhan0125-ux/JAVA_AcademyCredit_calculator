# 成大畢業學分計算器 (NCKU Graduation Credit Calculator)

## 專案簡介
本專案為一套以 Java 開發的「畢業學分計算器」，協助資訊工程系學生快速了解自己的畢業進度，自動統計已修學分、各類別學分以及畢業資格狀態。

本專案除了實作功能外，也希望實踐軟體工程相關概念，包括：
- 軟體架構設計 (Software Architecture)
- Clean Code
- Unit Testing
- Test Driven Development (TDD)
- Debugging
- Refactoring
- AI Assisted Development

---

## 課程分類與代碼規則

輸入課程代碼後，系統依以下規則自動判斷類別：

| 課程代碼規則 | 分類 | 處理方式 |
|---|---|---|
| `F7` 開頭 | 系內課程（必修/選修） | **自動查詢課程目錄**，課程名稱與類別由資料庫決定；不在目錄中則顯示錯誤 |
| `A9` 開頭 | 通識課程 | 選擇子類別（一般通識五類 / 融合通識）＋輸入學分數，名稱自動設為課程代碼 |
| `A7` 開頭 | 國文（校訂必修） | 僅輸入學分（上限 4 學分），名稱自動設為課程代碼 |
| `A1-100` ~ `A1-300` | 英文（校訂必修） | 僅輸入學分（上限 4 學分），名稱自動設為課程代碼 |
| `A1-500` ~ `A1-590` | 第二外語 | 無需任何輸入，固定 2 學分，名稱自動設為課程代碼 |
| `A2` 開頭 | 體育（校訂必修） | 選擇子類別（健康知能 / 專項體能 / 二年級），固定 0 學分，名稱自動設為課程代碼 |
| 其他 | 系外選修 | **唯一需手動輸入名稱**的類型；輸入名稱與學分數；若名稱與系內必修相同則詢問是否抵免 |

> **注意**：所有學分數支援 **0.5 為單位**（如 0.5、1.0、1.5、2.5 等）。輸入非 0.5 倍數的值將被拒絕。

### 體育修課規範
- 健康知能：至少 1 次
- 專項體能：至少 1 次
- 二年級體育：至少 2 次

---
### 通識修課規範
- 五類通識至少修習 3 個類別，且一般通識至少 4 學分
- 融合通識最少 1 學分，最多 15 學分
- 通識總學分（一般通識 + 融合通識）至少 18 學分

## 畢業門檻（資訊工程學系）

| 類別 | 需求說明 |
|---|---|
| 系上必修 | 60 學分（全部必修） |
| 系上選修 | 至少 21 學分 |
| 通識課程 | 18 學分（含一般通識、融合通識） |
| 系外選修 | 上限 21 學分（含第二外語） |
| 校訂必修—國文 | 4 學分 |
| 校訂必修—英文 | 4 學分 |
| 校訂必修—體育 | 健康知能×1 + 專項體能×1 + 二年級×2 |
| 校訂必修—踏溯台南 | 1 學分 |

**畢業總學分：130**

## 系統功能規劃

### ✅ 功能一：選單主介面
主畫面顯示各功能選單（1～9），選項 9 可快速載入模擬範例學生。

### ✅ 功能二：新增已修課程
- 輸入課程代碼後，系統優先查詢課程目錄，找到後**直接新增**（名稱、類別、學分全部自動帶入）
- 不在目錄時依代碼規則引導，輸入規則如下：
  - **F7（系內）**：必須存在於課程目錄，否則顯示錯誤提示；不支援手動分類
  - **A9（通識）**：選擇通識子類別 + 輸入學分數；課程名稱自動設為課程代碼
  - **A7（國文）**：僅輸入學分數（上限 4）；課程名稱自動設為課程代碼
  - **A1-100~300（英文）**：僅輸入學分數（上限 4）；課程名稱自動設為課程代碼
  - **A1-500~590（第二外語）**：固定 2 學分，無需任何輸入；課程名稱自動設為課程代碼
  - **A2（體育）**：選擇體育子類別（健康知能 / 專項體能 / 二年級）；固定 0 學分，名稱自動設為課程代碼
  - **其他（系外選修）**：需手動輸入課程名稱與學分數；若名稱與系上必修相同則詢問是否申請抵免
- **學分數以 0.5 為單位**（如 0.5、1.5、2.5）；輸入非 0.5 倍數的值系統將提示重新輸入

### ✅ 功能三：課程清單分類輸出
依課程類別分組顯示所有已修課程，例如：
```
【系上必修】（51 學分）
  - [F74401] 程式設計（3 學分）
  - [F74402] 資料結構（3 學分）

【通識—融合通識】（8 學分）
  - [A9F001] 科技與人文（4 學分）

【體育—健康知能】（1 次）
  - [A2-001] 健康與體育知識
```

### ✅ 功能四：分類統計學分 ＋ 達上限提醒
依課程細分類統計學分，在選單 4（分類統計）顯示所有有修習的類別。

### ✅ 功能五：顯示各類別剩餘學分
選單 5 顯示各類別尚缺學分（或體育次數），僅顯示尚有缺額的項目。

### ✅ 功能六：判斷畢業資格
選單 6 顯示是否符合畢業資格。

### ✅ 功能七：產生畢業進度報告
選單 7 產生格式化的畢業進度報告，包含：
```
╔══════════════════════════════════════════╗
║          畢業進度報告                    ║
╚══════════════════════════════════════════╝
  學生姓名：王小明
  總學分  ：115 / 130（88%）
──────────────────────────────────────────
  系上必修    ：55 / 60 學分  ✗ 尚缺 5
  系上選修    ：27 / 21 學分  ✓ 已達標
  通識（總計）：16 / 18 學分  ✗ 尚缺 2
  ...
  體育—二年級 ：1 / 2 次  ✗ 尚缺 1 次
──────────────────────────────────────────
  畢業狀態：❌ 尚未符合畢業資格
```

### ✅ 功能八：查看課程目錄
選單 8 顯示完整的 NCKU 格式課程目錄（依類別分組）。

### ✅ 功能九：模擬範例學生
選單 9 載入預設範例學生「王小明」，展示部分完成的畢業進度。

## 資料儲存

### ✅ 本機資料保存（記憶體 In-Memory）
課程目錄使用 `InMemoryCourseRepository` 儲存，啟動時自動載入 NCKU 格式課程目錄。

> **預留擴充**：介面 `CourseRepository` 已設計為可抽換，未來只需新增 `JsonCourseRepository` 或 `JdbcCourseRepository` 即可持久化，Service 層無需修改。

### ⬜ 資訊系課程資料庫（人工建入）
將資訊工程系的所有課程資料手動建入，提供代碼查詢（不須AI實作）。

## 系統架構

採用分層架構 (Layered Architecture)：

### Model 層
- `Student` — 學生資料與已修課程清單
- `Course` — 課程資料（代碼、名稱、學分、分類）
- `GraduationRequirement` — 畢業門檻設定（含通識子規則、體育次數規則）
- `GraduationReport` — 畢業進度報告（含 PE 次數、通識加總）
- `Enum CreditCategory` — 16 個細分類：
  - 系必、系選
  - 通識_人文、通識_社科、通識_自然、通識_生命、通識_科際、通識_融合
  - 系外選修、第二外語
  - 國文、英文
  - 體育_健康、體育_專項、體育_二年級
  - 踏溯台南

### Service 層
- `CourseClassifier` — 依 F7/A9/A7/A1/A2 課程代碼規則自動分類
- `CreditCalculator` — 學分計算、次數統計
- `GraduationChecker` — 畢業資格審核（含複雜通識聚合規則）

### Repository 層
- `CourseRepository`（介面）— 可抽換的資料存取介面
- `InMemoryCourseRepository` — 記憶體實作，啟動時載入 NCKU 課程目錄

## UML 初步規劃

### Student
- studentId
- studentName
- completedCourses

### Course
- courseId
- courseName
- credits（**double**，支援 0.5 倍數，如 0.5 / 1.0 / 1.5 / 2.5）
- category（CreditCategory）
- `formatCredits(double)` — 靜態工具方法，整數顯示為 `3`，半學分顯示為 `1.5`

### GraduationRequirement
- totalCreditsRequired（130）
- requiredMajorMin / majorElectiveMin
- generalTotalMin / generalCategoryMinCount / generalFusionMin / generalFusionMax
- chineseCredits / englishCredits / introTainanCredits
- peHealthMin / peSportMin / peSophomoreMin

### CourseClassifier
- detectType(courseId) → CodeType
- classifyDirect(courseId) → CreditCategory

### CreditCalculator
- `calculateTotalCredits()` → `double`
- `calculateCreditsByCategory()` → `Map<CreditCategory, Double>`
- `countCoursesByCategory()` → `Map<CreditCategory, Integer>`（次數仍為整數）

### GraduationChecker
- checkGraduation()
- calculateShortfall()
- generateReport()

## 單元測試需求（JUnit 5）

已涵蓋以下測試案例：

2. ✅ 0.5 學分課程加總正確（TC04）
