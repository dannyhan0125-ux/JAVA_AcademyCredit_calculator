# 成大畢業學分計算器 (NCKU Graduation Credit Calculator)

## 專案簡介
本專案為一套以 Java 開發的「畢業學分計算器」，協助資訊工程系學生快速了解自己的畢業進度，自動統計已修學分、各類別學分以及畢業資格狀態。

本專案除了實作功能外，也希望實踐軟體工程相關概念，包括：
- 軟體架構設計 (Software Architecture)
- Clean Code
- Unit Testing
- Test Driven Development (TDD)
- Debugging
- Refactoring
- AI Assisted Development

---

## 課程分類與代碼規則

輸入課程代碼後，系統依以下規則自動判斷類別：

| 課程代碼規則 | 分類 | 處理方式 |
|---|---|---|
| `F7` 開頭 | 系內課程（必修/選修） | 直接加上學分 |
| `A9` 開頭 | 通識課程 | 選擇類別（一般通識五類 / 融合通識），輸入學分數 |
| `A7` 開頭 | 國文（校訂必修） | 直接加入，上限 4 學分 |
| `A1-100` ~ `A1-300` | 英文（校訂必修） | 直接加入，上限 4 學分 |
| `A1-500` ~ `A1-590` | 第二外語 | 固定 2 學分，加入系外選修 |
| `A2` 開頭 | 體育（校訂必修） | 選擇學期（健康知能 / 專項體能 / 二年級），需各達修課要求 |
| 其他 | 系外選修 | 手動輸入學分數 |

### 體育修課規範
- 健康知能：至少 1 次
- 專項體能：至少 1 次
- 二年級體育：至少 2 次

## 畢業門檻（資訊工程學系）

| 類別 | 需求說明 |
|---|---|
| 系上必修 | 60 學分（全部必修） |
| 系上選修 | 至少 21 學分 |
| 通識課程 | 19 學分（含一般通識、融合通識） |
| 系外選修 | 上限 21 學分 |
| 校訂必修—國文 | 4 學分 |
| 校訂必修—英文 | 4 學分 |
| 校訂必修—體育 | 健康知能×1 + 專項體能×1 + 二年級×2 |
| 校訂必修—大學入門 | 1 學分 |

**畢業總學分：130**

## 系統功能規劃

### ⬜ 功能一：選單主介面
主畫面啟動時顯示目前各類別學分狀態與到達上限的提醒。

### ⬜ 功能二：新增已修課程
- 按下「新增」後進入新增介面
- 選擇課程類型：**必修 / 選修 / 通識**
- 輸入課程代碼後，系統自動依代碼規則判斷分類，並引導輸入對應資訊

### ⬜ 功能三：課程清單分類輸出
依課程類別分組顯示所有已修課程，例如：
```
【系上必修】
  - 資料結構 (3 學分)
  - 演算法 (3 學分)

【通識—融合通識】
  - 科技與人文 (2 學分)
```

### ⬜ 功能四：分類統計學分 ＋ 達上限提醒
依課程類別統計學分，並在達到上限時主動提醒：
- 系外選修達 21 學分上限 → 提醒
- 融合通識 / 一般通識達上限 → 提醒
- 英文 / 國文 / 體育達到應修要求 → 提醒

每次開啟應用程式時，於主畫面顯示上述提醒。

### ✅ 功能五：顯示各類別剩餘學分
- 校訂必修（國文）剩餘：... 學分
- 校訂必修（英文）剩餘：... 學分
- ⬜校訂必修（體育）：健康知能/專項體能/二年級 各別狀態
- 系上選修剩餘：... 學分
- 系上選修尚缺：... 學分

### ⬜ 功能七：產生畢業進度報告
範例：
```
學生：王小明
總學分：102/130
系上必修：55/60
系上選修：21/21（已達下限）
通識：15/19
  └ 一般通識：10 學分
  └ 融合通識：5 學分
系外選修：12/21
校訂必修—國文：4/4 ✓
校訂必修—英文：3/4
校訂必修—體育：健康知能✓ 專項✓ 二年級 1/2
校訂必修—踏溯台南：1/1 ✓
畢業狀態：尚未符合畢業資格
```

## 資料儲存

### ⬜ 本機資料保存
課程資料儲存於本機（如 JSON 或純文字檔），下次開啟時自動載入，不需重新輸入。

### ⬜ 資訊系課程資料庫
將資訊工程系的所有課程資料（課程代碼、名稱、學分數、類別）手動建入 `Main.java` 或對應的 Repository，提供代碼查詢與自動補全功能。

## 系統架構

採用分層架構 (Layered Architecture)：

### Model 層
- `Student`
- `Course`
- `GraduationRequirement`
- `GraduationReport`
- `Enum CreditCategory`（含：系必、系選、通識_一般、通識_融合、系外選修、國文、英文、第二外語、體育_健康、體育_專項、體育_二年級、大學入門）

### Service 層
- `CreditCalculator`
- `GraduationChecker`

### Repository 層
- `CourseRepository`（本機儲存）
- `DepartmentCourseDatabase`（資訊系課程資料庫）

## Clean Code 要求
- 使用有意義的命名
- Method 不宜過長
- 一個 Method 只負責一件事
- 避免重複程式碼
- 避免 Main 方法過於肥大
- 遵守 Single Responsibility Principle

## 單元測試需求（JUnit 5）

至少包含以下測試案例：
1. 正確計算總學分
2. 正確計算分類學分
3. 正確計算缺少學分
4. 符合畢業資格
5. 不符合畢業資格
6. 超過門檻時缺額應為 0
7. 各類型課程代碼判斷正確
8. 體育修課次數判斷

---

## TDD 開發流程
遵循：Red -> Green -> Refactor
1. 先撰寫測試案例
2. 執行測試（失敗）
3. 撰寫最小可行程式碼
4. 讓測試通過
5. 重構程式碼
6. 再次執行所有測試

---

## Debugging 流程

2. 提出假設
3. 設計驗證方法
4. 驗證假設
5. 修正根本原因
